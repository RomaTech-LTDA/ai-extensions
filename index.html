@romatech/ai-extensions

GitHub npm

Plug-and-play AI enablement framework for Node.js APIs. Transforms your endpoints into MCP tools and RAG-enabled knowledge sources — identical to the .NET version.

Works with Express, Fastify, Koa, and Hono.

Quick Start

import 'reflect-metadata';
import express from 'express';
import {
    Controller, Get, Post, Delete,
    AiTool, AiHidden, AiDescription,
    useController, useAi
} from '@romatech/ai-extensions';

@Controller('/api/orders')
class OrdersController {
    @Get('/')
    @AiDescription('Lists all orders')
    static getAll(req, res) {
        res.json([{ id: 1, status: 'Pending' }]);
    }

    @Post('/')
    @AiTool('create_order')
    @AiDescription('Creates a new order')
    static create(req, res) {
        res.status(201).json({ id: 2, ...req.body });
    }

    @Delete('/:id')
    @AiHidden()
    static delete(req, res) {
        res.sendStatus(204);
    }
}

const app = express();
app.use(express.json());

useController(app, OrdersController);
useAi(app, { baseUrl: 'http://localhost:3000' });

app.listen(3000);

Features

  • Decorator-based — @AiTool, @AiHidden, @AiDescription (like .NET attributes)
  • Multi-framework — Express, Fastify, Koa, Hono, NestJS, Next.js, tRPC, Lambda (auto-detected)
  • MCP Server — Full Model Context Protocol (tools, resources, prompts, completions)
  • RAG Search — Semantic search via built-in rag_search tool
  • Swagger Discovery — Auto-discovers endpoints from OpenAPI spec
  • Custom Handlers — Bypass HTTP with in-process tool handlers
  • Tool Versioning — v1/v2 with deprecation support
  • Dry-run Mode — Validate without executing
  • Auth & CORS — API key auth + configurable CORS
  • Metrics + Audit — Tool usage tracking + execution history
  • Circuit Breaker — Auto-disable failing tools
  • Multi-tenant — Different tools per API key
  • SSE + WebSocket — Persistent connections with heartbeat
  • Embedding Providers — Local, OpenAI, Ollama
  • Zero External Deps — Core works without OpenAI/Ollama

// Hono
useController(app, MyController);

MCP Protocol

# List tools
POST /mcp  {"jsonrpc":"2.0","id":1,"method":"tools/list"}

# Call a tool
POST /mcp  {"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"create_order","arguments":{...}}}

# RAG search
POST /mcp  {"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"rag_search","arguments":{"query":"..."}}}

# Health check
GET /mcp/health

Architecture

Your API (Express/Fastify/Koa/Hono)
       ↓
Controller Decorators (@AiTool, @AiHidden, @Get, @Post)
       ↓
useController() → registers routes + AI metadata
       ↓
useAi() → Swagger Discovery + MCP + RAG
       ↓
MCP Clients (Claude, GPT, Kiro, etc.)
